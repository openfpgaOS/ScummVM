# ScummVM Port — Changes to Revert / Clean Up

Workarounds and hacks applied to get ScummVM compiling and running on
openfpgaOS. These should be revisited as the port matures.

## portdefs.h
- `#define dynamic_cast reinterpret_cast` — Dangerous global macro to work around `-fno-rtti`. Should be replaced with either enabling RTTI (`-frtti`) or patching the ~5 ScummVM files that use `dynamic_cast` to use type tags instead.

## compat/zlib_stub.cpp
- All zlib/inflate functions are stubbed (return error). Games that need compressed resources (COMI v7) won't work. Should vendor miniz or tinfl for inflate-only support.
- `SVGBitmap` constructor creates a dummy empty surface. GUI icons won't render.

## config.h
- `STATIC_PLUGIN 1` / `DYNAMIC_MODULES 0` — Hardcoded. Should align with ScummVM's actual plugin system.
- Many `#undef USE_*` for disabled features — Fine, but should be regenerated if we add features.
- `SCUMMVM_VERSION` is hardcoded — Should track the vendored ScummVM version.

## Makefile
- Source files discovered via `$(wildcard)` with exclusion filters — Fragile. May break when ScummVM is updated. Should eventually use ScummVM's own module.mk system.
- Excluded files: `recorderfile.cpp`, `updates.cpp`, `alsa_opl.cpp`, `opl2lpt.cpp`, `retrowave_opl3.cpp`, `rwopl3.cpp`, `fluidsynth.cpp`, `mt32.cpp`, `eas.cpp`, `ac3.cpp`, `larryScale*.cpp`, `svg.cpp`, `blit-atari.cpp`, `blit-neon.cpp`, `blit-sse2.cpp`, `blit-avx2.cpp`, `bink_decoder.cpp`, `hnm_decoder.cpp`, `mpegps_decoder.cpp`, `psx_decoder.cpp`, `mkv_decoder.cpp`, `theora_decoder.cpp`, `gif.cpp`, `mpeg.cpp`, `onscreendialog.cpp`, `recorderdialog.cpp`, `updates-dialog.cpp`, `EventRecorder.cpp`, `editrecorddialog.cpp`, `fluidsynth-dialog.cpp`, `dump-all-dialogs.cpp`, `integrity-dialog.cpp`, `dialog-createsession.cpp`, `dialog-sessionselector.cpp`, `nut_renderer.cpp`, `string_v7.cpp`, `script_v8.cpp`.
- SCUMM HE directory partially included (`he/mixer_he.cpp`, `he/resource_he.cpp`, `he/script_v60he.cpp`, `he/script_v70he.cpp`, `he/sound_he.cpp`) — These are needed by the base SCUMM engine but ideally should be behind `ENABLE_HE` guards.

## backend/openfpga_osystem.cpp
- `initBackend()` has an idempotent guard (`if (_ofGfx) return`) because `scummvm_main()` calls it again. This is a hack — should properly separate first-time init from re-init.
- `createConfigReadStream()` / `createConfigWriteStream()` return nullptr — Config is never saved. Should eventually write to an APF save slot.
- `getTimeAndDate()` returns a fixed date (2025-01-01 12:00) — No RTC available.
- `NullMixerManager` used — No audio output yet for sound effects/voice. Replace with a real mixer that calls `of_audio_stream_write()`.
- Mouse cursor compositing done in software on the framebuffer — could flicker. Consider double-buffering the cursor overlay.

## backend/openfpga_midi.cpp
- The MIDI driver bridges ScummVM events to `of_smp_voice_*` directly; it does not use `of_midi_play()` (which expects a complete `.mid` file). Volume / expression / filter / pan tracking is per-channel and complete enough for iMUSE; CC91/CC93 (reverb/chorus) are forwarded but the SDK currently no-ops them.
- Pitch-bend range is hardcoded to the device default — if a game depends on RPN pitch-bend-sensitivity, we'd need to honour `setPitchBendRange()`.
- A `.ofsf` SoundFont bank must be staged in a data slot for the driver to open. `open()` returns `MERR_DEVICE_NOT_AVAILABLE` if no bank is present and ScummVM falls back to its software OPL3 emulator (`audio/softsynth/opl/dosbox.cpp`).
- We register our own `PluginProvider` from `main.cpp` instead of patching `scummvm/base/plugins.cpp` to add a `LINK_PLUGIN(OPENFPGA_MIDI)`. Either approach works; the provider keeps the submodule unmodified.

## ScummVM vendored source (scummvm/ submodule)
- `engines/plugins_table.h` — Hand-written (normally auto-generated by configure).
- `engines/detection_table.h` — Hand-written.
- No other ScummVM source files are modified.

## main.cpp
- Game is hardcoded to Monkey Island 1 VGA floppy — Should detect the game from the loaded data slots.
- Data slot IDs are hardcoded — Should be driven by the instance JSON or auto-detected.
- `scummvm_main()` called with hardcoded argv — Should be configurable.
